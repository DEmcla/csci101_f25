Part 1: The History of Computer Networking

The Problem: Why Networking Was Hard

Before the 1960s, computers couldn't communicate with each other. The telephone system used circuit switching, which was inefficient for computers:

Circuit Switching (Old Way) Packet Switching (New Way)
Dedicated line between two points Shared network paths
Like a phone call - line is reserved Like sending postcards - each travels independently
Wasteful when idle Efficient use of bandwidth
Single point of failure Resilient to failures

The Solution: Packet Switching

Packet Switching: The revolutionary idea that made computer networking possible. Instead of dedicating a line between two points, data is broken into small pieces called packets that travel independently and are reassembled at the destination.

Key characteristics:

  • Proposed by Leonard Kleinrock (1961) using queuing theory
  • Messages broken into small packets
  • Each packet travels independently
  • Packets can take different routes
  • Reassembled at destination

Key Terms:

  • Packet: Small unit of data with header information
  • Routing: Finding the best path for packets
  • Queuing Theory: Mathematical foundation for packet switching

Key Milestones in Networking History

Year Event Significance
1961 Packet switching theory Leonard Kleinrock's mathematical foundation
1965 First long-distance connection MIT to California at 300 bps
1969 ARPANET goes live First 4 nodes: UCLA, SRI, UCSB, Utah
1971 Email invented First "killer app" of networking
1973 Ethernet invented Robert Metcalfe at Xerox PARC
1974 TCP created Vint Cerf and Bob Kahn
1978 TCP/IP split Separated into IP (routing) and TCP (reliability)
1983 ARPANET adopts TCP/IP "Flag Day" - the internet is born
1984 OSI model published ISO's 7-layer reference model

Key People in Networking History

Person Contribution
Leonard Kleinrock Packet switching theory (queuing theory)
Larry Roberts ARPANET project leader
Vint Cerf TCP/IP co-creator ("Father of the Internet")
Bob Kahn TCP/IP co-creator
Robert Metcalfe Ethernet inventor

The Standards War: Why TCP/IP Won

In the 1980s, multiple competing protocols existed:

  • ARPANET (TCP/IP)
  • IBM's SNA
  • Digital's DECnet
  • Xerox's XNS
  • Novell's IPX/SPX

The OSI Model (1984) was created by ISO as a theoretical 7-layer reference model, intended to be THE networking standard. It became a teaching model instead.

Why TCP/IP won:

  • Already working and deployed
  • Open and free (no licensing)
  • Simpler and more practical
  • ARPANET/Internet adoption drove it
  • "Running code wins"
Key Insight: Understanding WHY decisions were made helps you understand modern networking.

Part 2: Networking Models

The OSI Model (7 Layers)

OSI Model: The Open Systems Interconnection model is a theoretical framework that describes how data moves through a network. It has 7 layers, each with a specific function.

Layer Name Function Examples
7 Application What users interact with HTTP, HTTPS, FTP, SMTP, DNS, SSH
6 Presentation Data formatting and encryption SSL/TLS, compression, character encoding
5 Session Manages connections/sessions Session establishment, keeping you logged in
4 Transport Reliable (or fast) delivery TCP (reliable), UDP (fast)
3 Network Routing between networks IP (Internet Protocol), Routers
2 Data Link Communication on same network MAC addresses, Switches, Ethernet
1 Physical Actual signal transmission Cables, wireless signals, electrical impulses
Memory Tricks:

Top to Bottom (7 to 1): "All People Seem To Need Data Processing"

Bottom to Top (1 to 7): "Please Do Not Throw Sausage Pizza Away"

The TCP/IP Model (4 Layers)

TCP/IP Model: The practical model that actually runs on the internet. It combines some OSI layers into a simpler 4-layer structure.

TCP/IP Layer Function Key Protocols OSI Equivalent
Application User services HTTP, FTP, SMTP, DNS, SSH Layers 5-7
Transport End-to-end delivery TCP, UDP Layer 4
Internet Routing between networks IP, ICMP Layer 3
Network Interface Local network communication Ethernet, Wi-Fi Layers 1-2
OSI to TCP/IP Mapping: OSI Model TCP/IP Model +--------------+ +------------------+ | Application | | | +--------------+ | Application | | Presentation | --> | | +--------------+ +------------------+ | Session | +--------------+ | Transport | --> | Transport | +--------------+ +------------------+ | Network | --> | Internet | +--------------+ +------------------+ | Data Link | | | +--------------+ --> | Network Interface| | Physical | | | +--------------+ +------------------+

Why Both Models Exist:

  • OSI: Great for understanding concepts, teaching, troubleshooting
  • TCP/IP: What actually runs on the internet

Encapsulation

Encapsulation: The process where each layer adds its own header information as data moves down the stack. Data gets "wrapped" going down, "unwrapped" going up.

Layer Data Unit Name What's Added
Application Data Application info
Transport Segment Port numbers, sequence numbers
Network Packet IP addresses
Data Link Frame MAC addresses
Physical Bits Electrical/optical signals
Sending Email - Encapsulation Example: Application: [Email Content] Transport: [TCP Header][Email Content] (port 25) Network: [IP Header][TCP Header][Email Content] (source/dest IP) Data Link: [Frame Header][IP Header][TCP Header][Email Content][Trailer] (MAC addresses) Physical: 10110010110010110011... (bits on the wire)
Key Insight: Each layer only needs to understand its own header. This is why layers are independent and can be updated without affecting other layers.

Network Devices and Their Layers

Device Layer Uses Function
Router Network (3) IP addresses Connects DIFFERENT networks
Switch Data Link (2) MAC addresses Connects devices on SAME network
Hub Physical (1) N/A Dumb repeater (obsolete)
Memory trick:
  • Router = "Routes between networks" (like a post office sorting mail between cities)
  • Switch = "Switches within a network" (like a mail room directing to specific offices)

Part 3: Protocols

TCP vs UDP

The two main transport layer protocols have very different characteristics:

Feature TCP UDP
Connection Connection-oriented Connectionless
Reliability Guaranteed delivery Best effort (no guarantee)
Order Maintains packet order No ordering
Speed Slower (overhead) Faster (minimal overhead)
Handshake 3-way handshake None
Error Recovery Retransmits lost packets No retransmission
Analogy Certified mail Postcard

TCP Three-Way Handshake

1. SYN - Client: "Want to connect?" 2. SYN-ACK - Server: "Yes, I'm ready" 3. ACK - Client: "Great, let's go!"

When to Use Each Protocol

Use Case Protocol Why?
Web browsing TCP Need complete HTML
Email TCP Must arrive intact
File download TCP Every byte matters
Video call UDP Speed matters, old frames useless
Online gaming UDP Latency critical
DNS query UDP Simple, fast lookup
Video streaming UDP Buffering handles packet loss

Other Important Protocols

Protocol Layer Purpose
DNS Application Translates domain names to IP addresses (e.g., google.com → 142.250.80.78)
DHCP Application Automatically assigns IP addresses to devices
HTTP/HTTPS Application Web browsing (HTTPS = encrypted)
SMTP Application Sending email
FTP Application File transfer
SSH Application Secure remote access
ICMP Network Error messages and diagnostics (ping uses this)
ARP Data Link Maps IP addresses to MAC addresses

Part 4: IP Addressing

IPv4 Addressing

IPv4: Internet Protocol version 4. Uses 32-bit addresses written in dotted decimal notation.

  • Format: 192.168.1.1 (4 octets, dotted decimal)
  • Size: 32 bits (~4.3 billion addresses)
  • Problem: Running out of addresses!

Private vs Public IP Addresses

Type Range Use
Private 10.0.0.0 - 10.255.255.255 Large internal networks
Private 172.16.0.0 - 172.31.255.255 Medium internal networks
Private 192.168.0.0 - 192.168.255.255 Home/small office networks
Public Everything else Internet-routable addresses
Quick Check: If an IP starts with 10., 172.16-31., or 192.168., it's private.

NAT (Network Address Translation)

NAT: Translates private IP addresses to public IP addresses and back. Allows many devices to share one public IP address.

Why NAT exists:

  • IPv4 address exhaustion (only ~4.3 billion addresses)
  • Security (hides internal network structure)
  • Cost (don't need a public IP for every device)
Your laptop: 192.168.1.100 (private) | Home Router (NAT) | Internet sees: 74.125.224.72 (public - assigned by your ISP)

IPv6 Addressing

IPv6: Internet Protocol version 6. Uses 128-bit addresses written in hexadecimal with colons.

  • Format: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Size: 128 bits (practically unlimited addresses)
  • Shortening: 2001:db8:85a3::8a2e:370:7334 (:: replaces consecutive zeros)

IPv4 vs IPv6 Comparison

Feature IPv4 IPv6
Address size 32 bits 128 bits
Format Dotted decimal (192.168.1.1) Hexadecimal with colons
Total addresses ~4.3 billion ~340 undecillion
Header Variable length Fixed 40 bytes
Security Optional (IPSec) Built-in (IPSec required)
NAT Common/necessary Not needed

Part 5: Command Line Interface (CLI)

Why CLI Matters

  • Faster than clicking through GUIs
  • Can be automated with scripts
  • Works over remote connections (SSH)
  • Available on all systems
  • Essential for network troubleshooting

Navigation Commands

Task Windows Mac/Linux
Show current directory cd (no args) pwd
List files dir ls
Change directory cd folder cd folder
Go up one level cd .. cd ..
Go to home directory cd %USERPROFILE% cd ~
Clear screen cls clear

Path Concepts

Absolute Path: Complete path from root Windows: C:\Users\Student\Documents\file.txt Mac/Linux: /home/student/documents/file.txt Relative Path: Path from current location ./file.txt (file in current directory) ../file.txt (file in parent directory) folder/file.txt (file in subfolder)

Network Diagnostic Commands

ping - Test Connectivity

$ ping google.com Purpose: Tests if a host is reachable Uses: ICMP protocol Shows: Round-trip time (latency) Indicates: Packet loss = network problems

traceroute / tracert - Trace Path

$ traceroute google.com (Mac/Linux) $ tracert google.com (Windows) Purpose: Shows each hop (router) between you and destination Uses: Helps identify where problems occur Shows: Latency at each hop

ipconfig / ifconfig - Network Configuration

$ ipconfig (Windows) $ ifconfig (Mac/Linux - older) $ ip addr (Linux - modern) Shows: - Your IP address(es) - Subnet mask - Default gateway - DNS servers

netstat - Network Statistics

$ netstat -an Purpose: Shows network connections Shows: - Active connections - Listening ports - Connection states

nslookup / dig - DNS Lookup

$ nslookup google.com $ dig google.com (Mac/Linux) Purpose: Queries DNS servers Shows: IP address for domain name Uses: Troubleshoot DNS issues

File Operations

View file contents type file.txt cat file.txt
Create folder mkdir folder mkdir folder
Copy file copy src dest cp src dest
Move/rename move src dest mv src dest
Delete file del file.txt rm file.txt

Common Mistakes to Avoid

Things Students Often Get Wrong

  • Confusing OSI and TCP/IP models - OSI has 7 layers (theoretical), TCP/IP has 4 (practical)
  • Mixing up TCP and UDP - TCP is reliable/slow, UDP is unreliable/fast
  • Confusing routers and switches - Routers use IPs (Layer 3), switches use MACs (Layer 2)
  • Forgetting layer numbers - Use the mnemonic: "All People Seem To Need Data Processing"
  • Private vs Public IPs - Remember: 10.x, 172.16-31.x, 192.168.x are always private
  • Encapsulation order - Data → Segment → Packet → Frame → Bits (going down)

Practice Questions

Test Your Understanding

1. What technology did packet switching replace?

Answer: Circuit switching

2. How many layers does the OSI model have? TCP/IP model?

Answer: OSI = 7 layers, TCP/IP = 4 layers

3. Which protocol is reliable - TCP or UDP?

Answer: TCP

4. What does DNS do?

Answer: Translates domain names to IP addresses

5. What command tests if a host is reachable?

Answer: ping

6. At which layer do routers operate?

Answer: Network layer (Layer 3)

7. What does the three-way handshake establish?

Answer: A TCP connection (SYN, SYN-ACK, ACK)

8. Is 10.0.0.50 a public or private IP address?

Answer: Private (10.x.x.x is always private)

9. What protocol would you use for live video streaming?

Answer: UDP (speed over reliability)

10. What does NAT stand for and what does it do?

Answer: Network Address Translation; translates private IPs to public IPs

11. Why did TCP/IP win over the OSI protocols?

Answer: Already deployed, open/free, simpler, practical ("running code wins")

12. What's the difference between a router and a switch?

Answer: Router connects different networks using IP addresses (Layer 3); switch connects devices on the same network using MAC addresses (Layer 2)

13. Put these in encapsulation order (sending): Packet, Bit, Frame, Segment

Answer: Segment → Packet → Frame → Bit

14. What information does traceroute show that ping doesn't?

Answer: The path/hops between source and destination

15. Name two differences between IPv4 and IPv6.

Answer: Any two of: address size (32 vs 128 bits), format (dotted decimal vs hex/colons), address space, built-in security in IPv6, no NAT needed for IPv6
