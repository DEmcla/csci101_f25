T / F
CORRECT ANSWER: TRUE - ARPANET was designed with distributed architecture and packet switching to create a resilient network that could survive node failures, which was a key requirement for military and research applications.

IPv6 was developed primarily to address the IPv4 address exhaustion problem and uses 128-bit addresses.
T / F
CORRECT ANSWER: TRUE - IPv6 uses 128-bit addresses (providing 340 undecillion addresses) and was developed to solve IPv4 address exhaustion.
Question 8 2 points
The Data Link Layer (Layer 2) is responsible for routing packets across multiple networks.
T / F
CORRECT ANSWER: FALSE - The Network Layer (Layer 3) is responsible for routing across networks. The Data Link Layer provides reliable communication between directly connected devices.
Question 9 2 points
Circuit switching is more efficient than packet switching for bursty computer data communications.
T / F
CORRECT ANSWER: FALSE - Packet switching is more efficient for bursty data because it allows multiple communications to share network resources. Circuit switching reserves a dedicated path that sits idle during pauses.
Question 10 2 points
Traditional shared-medium Ethernet uses CSMA/CD (Carrier Sense Multiple Access with Collision Detection) to manage access to the network and handle collisions.
T / F
CORRECT ANSWER: TRUE - CSMA/CD was the protocol used in early shared-medium Ethernet (hubs, coaxial cable) to coordinate access and handle collisions. Modern switched, full-duplex Ethernet does not require CSMA/CD.

Question 26 4 points
In this module, we discussed how TCP establishes reliable connections before data transfer.

a) List the three messages exchanged during TCP's three-way handshake in order. (1.5 points)

b) Explain why this process is called a "handshake" - what is being agreed upon between the two devices? (1.5 points)

c) What would happen if the third message (ACK) was lost due to network congestion? (1 point)
[Student answer space]
SAMPLE ANSWER:

a) The three messages in order:

  1. SYN (Synchronize)
  2. SYN-ACK (Synchronize-Acknowledge)
  3. ACK (Acknowledge)

b) It's called a "handshake" because both devices agree on initial sequence numbers and confirm they are ready to communicate reliably. They establish parameters for the connection before exchanging actual data.

c) If the ACK is lost, the server doesn't know the client received its SYN-ACK. The server will wait for the ACK, eventually timeout, and may retransmit the SYN-ACK. The connection won't be fully established until the server receives the ACK.

Feature 1: A live video stream that broadcasts a lecturer speaking to 500 students

Question 28 3 points
A student says: "Network layering is just unnecessary complexity. A single monolithic protocol would be simpler and faster."

Respond to this student by:
a) Identifying one reason why they might think this (1 point)

b) Explaining two benefits of layering that prove why this claim is incorrect (2 points)
[Student answer space]
SAMPLE ANSWER:

a) They might think this because layering does add some overhead (each layer adds headers) and requires coordination between layers, which can seem like extra complexity compared to a single unified system.

b) Two benefits that prove layering is actually better:

  • Modularity/Flexibility: You can upgrade or replace one layer without changing others. For example, you can switch from copper Ethernet to fiber optic at the Physical layer without changing any applications. A monolithic protocol would require complete replacement for any upgrade.
  • Troubleshooting: Problems can be isolated to specific layers. If you can't reach a website, you can systematically check: Physical (cables?), Network (IP config?), Transport (TCP?), Application (HTTP?). With a monolithic protocol, you'd have to debug the entire system at once.

Alternative benefits: Interoperability (different vendors can build compatible equipment), Abstraction (layers hide complexity), Specialization (each layer focuses on one task).

Grading Rubric:
• 1 point for part (a) - valid reason student might think layering is complex (overhead, coordination, abstraction)
• 2 points for part (b) - two valid benefits with explanations (1 point each: modularity, troubleshooting, interoperability, abstraction, flexibility)

Consider the following two devices on a network:

Device A: IP address 192.168.10.75/26
Device B: IP address 192.168.10.135/26

a) Are these two devices on the same subnet or different subnets? (1 point)

b) Show your work by determining the network address for each device. (2 points)

c) What is the subnet mask in dotted decimal notation for this network? (1 point)
[Student completes on paper]
SAMPLE ANSWER:

a) Answer: Different subnets

b) Work shown:

Step 1: Convert /26 to subnet mask

  • /26 means 26 bits are set to 1 in the subnet mask
  • 11111111.11111111.11111111.11000000
  • 255.255.255.192

Step 2: Determine network address for Device A (192.168.10.75)

  • Fourth octet: 75 in binary = 01001011
  • Mask: 11000000 (192)
  • Network bits (AND operation): 01000000 = 64
  • Network address: 192.168.10.64

Step 3: Determine network address for Device B (192.168.10.135)

  • Fourth octet: 135 in binary = 10000111
  • Mask: 11000000 (192)
  • Network bits (AND operation): 10000000 = 128
  • Network address: 192.168.10.128

Conclusion: Device A is on network 192.168.10.64/26, Device B is on network 192.168.10.128/26. Since these are different network addresses, the devices are on different subnets.

c) Subnet mask in dotted decimal: 255.255.255.192

[Student answer space]
SAMPLE ANSWER:

a) Packet switching vs. circuit switching:

Packet switching breaks data into small packets that travel independently through the network and are reassembled at the destination. Circuit switching establishes a dedicated path between sender and receiver for the entire communication session. The key difference is that packet switching allows multiple users to share network resources dynamically, while circuit switching reserves bandwidth exclusively for one connection.

b) Personal scenario - Netflix + social media:

With packet switching, my Netflix stream and social media browsing can share my internet connection efficiently. When I pause to read a social media post, the Netflix packets slow down or stop, freeing up bandwidth. When I scroll past a video, those packets get priority. If I had to use circuit switching, I'd need two separate dedicated connections—one for Netflix and one for browsing—which would waste enormous amounts of bandwidth during idle moments. Packet switching allows both activities to dynamically share my limited home bandwidth, sending packets from whichever application needs them at any given moment.

Alternative acceptable answer: "Packet switching is perfect for this because my internet usage is 'bursty.' I'm not constantly downloading from Netflix—there are pauses when the buffer fills. Similarly, social media loads in bursts when I click links. Packet switching efficiently uses my bandwidth during these bursts. Circuit switching would reserve a constant connection for each activity even when nothing is being transmitted, wasting most of my available bandwidth."

SAMPLE ANSWER:

a) Table of 8 possible subnet mask values:

Binary (8 bits) Decimal
10000000 128
11000000 192
11100000 224
11110000 240
11111000 248
11111100 252
11111110 254
11111111 255

b) Why these are the ONLY valid values:

Subnet masks must consist of contiguous 1s followed by contiguous 0s (or all 1s). You cannot have a pattern like 10101010 or 11011000 because subnet masks identify the network portion (1s) and host portion (0s) of an IP address - these portions must be continuous blocks. The 1s must always come first, followed by 0s.

Alternative acceptable answer: "Subnet masks work by using a bitwise AND operation with IP addresses. The 1s identify which bits belong to the network address, and 0s identify host bits. To properly separate network and host portions, all network bits (1s) must be together on the left, and all host bits (0s) must be together on the right. This creates exactly 8 valid patterns for any octet (or 9 if you count 00000000, but that's not used in practice)."

You're helping a friend troubleshoot their internet connection. You run ping google.com and notice:
  • Some packets are getting through (60% success rate)
  • Response times vary wildly (20ms to 500ms)

Ping only tests the final destination endpoint. With tracert/traceroute, you can identify exactly which router or network segment is causing delays or packet loss.

Alternative acceptable answer for part a: "Yes, because ping shows a problem exists (packet loss, variable latency), but doesn't tell you where the problem is.
